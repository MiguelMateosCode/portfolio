THE ADECCO GROUP

Sistema de Diseño de The Adecco Group

El sistema de diseño de The Adecco Group unifica el aspecto visual de todas sus plataformas digitales.

Introducción

El contexto

The Adecco Group necesitaba que todos sus productos digitales tuvieran el mismo aspecto y funcionaran de forma coherente. Para lograrlo, creamos TAG DS: el sistema de diseño del grupo.

¿Qué es TAG DS? Es una biblioteca de componentes visuales —botones, formularios, colores, tipografías— que todos los equipos pueden usar. Así, todos los productos del grupo tienen una imagen consistente.

TAG DS se aplica en plataformas de marcas como Adecco, Akkodis y LHH.

Sus tres objetivos principales eran:

  • Hacer que todos los productos tengan el mismo aspecto visual
  • Mejorar la usabilidad (que sean fáciles de usar)
  • Mejorar la accesibilidad (que puedan usarlos todas las personas)

Retos

Cada equipo había construido su plataforma de forma independiente. Esto generaba tres problemas claros:

  • Cada plataforma tenía un aspecto diferente
  • Los estándares de accesibilidad variaban entre productos
  • Era difícil hacer una auditoría completa del estado de cada plataforma

Mi rol

Trabajé como diseñador de producto y colaborador del sistema de diseño.

Mi trabajo fue coordinar a diseñadores, desarrolladores y responsables de negocio. Participé en todo el proceso: desde la fase de descubrimiento hasta la construcción final.

También me encargué de documentar el sistema y publicarlo en Zeroheight, una herramienta de documentación para equipos de diseño.

Proceso de diseño

Investigación y análisis

Empezamos investigando a fondo antes de diseñar nada. Hicimos:

  • Entrevistas con los responsables de cada área del negocio
  • Análisis de sistemas de diseño de empresas competidoras
  • Talleres con personas usuarias para entender sus necesidades y problemas

Principios de diseño

Definimos tres principios que guiaron todas las decisiones:

  • Accesibilidad: el sistema debe poder usarlo cualquier persona
  • Reutilización: cada componente se diseña una vez y se usa muchas veces
  • Simplicidad: menos es más

Herramientas usadas

  • Figma: para diseñar y crear prototipos de forma colaborativa
  • Zeroheight: para documentar los componentes del sistema
  • Storybook: para compartir los componentes con el equipo de desarrollo

Creación del componente Time Picker

¿Por qué este componente?

¿Qué es un Time Picker? Es el selector que usas para introducir una hora en una aplicación. Por ejemplo, cuando reservas una cita o programas una reunión.

Las plataformas del grupo no tenían una forma coherente de introducir horas. Cada equipo lo había resuelto a su manera. Esto generaba una experiencia fragmentada para las personas usuarias.

Cómo lo diseñamos

El proceso tuvo varias fases:

  • Dibujamos bocetos y esquemas iniciales (wireframes)
  • Creamos prototipos interactivos en Figma
  • Realizamos pruebas con personas usuarias reales
  • Iteramos el diseño según el feedback recibido
  • Colaboramos con desarrollo para asegurar una implementación correcta

Estándares de accesibilidad

Comprobamos que el componente cumplía con los estándares WCAG (normas internacionales de accesibilidad web). También verificamos que funcionara bien integrado con el resto del sistema.

Resultados e impacto

Adopción del sistema

Desde su lanzamiento, TAG DS ha sido adoptado por los equipos de producto. El Time Picker ha recibido buenas valoraciones por su facilidad de uso y accesibilidad.

Mejora de la usabilidad

El sistema mejoró la puntuación de usabilidad de forma significativa:

Versión Puntuación de usabilidad
TAG DS V.0 (inicial) 45%
TAG DS V.1 (actual) 72%

Una mejora de 27 puntos porcentuales.

Lo que aprendimos

  • La colaboración entre distintos perfiles (diseño, desarrollo, negocio) es fundamental
  • El feedback de las personas usuarias debe guiar las decisiones de diseño
  • Una documentación clara facilita que el sistema sea adoptado y mantenido

El futuro del sistema

El sistema sigue evolucionando. Los próximos pasos incluyen:

  • Ampliar la biblioteca de componentes
  • Mejorar las guías de uso
  • Adaptarse a nuevas tecnologías y necesidades

Conclusión

Crear TAG DS fue un trabajo de colaboración e innovación. Establecimos un nuevo estándar de diseño en The Adecco Group. Demostramos que un sistema de diseño bien hecho transforma la experiencia de usuario y hace más eficiente el desarrollo de producto.
